01 — The Problem

Why We're Building This

Traditional process mining tools like Celonis and Signavio work well for ERP-based processes where one order ID travels through one system's audit log. Banking is fundamentally different.

A single mortgage journey fragments across 10–12 systems — CRM, LOS, ID Verification, Credit Bureau, Document Management, Valuation, Decision Engine, E-Sign, Conveyancing, Core Banking, Payments, Land Registry — each generating its own ID with no shared key. No existing tool handles this.

⏱

2-Day Workshops → 2 Hours

Replace manual discovery workshops where consultants whiteboard processes by hand with automated analysis from raw system exports.

🔗

Cross-System Correlation

Our Case Correlation Engine links events across 10+ systems using temporal proximity, customer spine, and cross-reference fields — no shared ID needed.

🕳

Dark Process Detection

Identify manual work happening between system timestamps — the "dark process zones" where Excel, phone calls, and email hide inefficiency.

🤖

APA Opportunity Output

Automatically surface which steps are candidates for Backbase APA agents — IDP, Flow 2.0, decision automation — with evidence from the data.

04 — Database

Data Models

All data lives in Neon PostgreSQL. The schema is managed by Prisma and version controlled.

User

Backbase team members. Email + hashed password. Role: admin or analyst.

Engagement

One discovery project per client. Stores process template, saved canvas map (JSON), status.

Upload

CSV files uploaded per engagement. Stores inferred schema, raw data, system source.

EventLog

Correlated events reconstructed from uploads. Case ID, activity, timestamp, actor, system.

AnalysisResult

AI mining output — bottlenecks, cycle times, rework loops, APA opportunities.

Config Tables

ProcessStepTemplate, ApplicationSystem, ProcessActor — all DB-driven and admin-configurable.

05 — Data Flow

How a Discovery Engagement Works

1

Create Engagement

Consultant creates an engagement (e.g. "BECU — Mortgage Discovery"), picks a process template. Templates are DB-driven — new ones can be generated by the AI Research Agent.

2

Model the Process

Consultant opens the BPMN Process Modeller canvas. Drags Start Event, Task nodes (pre-populated from DB), Gateways, End Event. Assigns actors and systems per step. Saves the process map.

3

Upload CSV Data

Consultant uploads raw CSV exports from the bank's operational systems (LOS, CRM, Core Banking, etc.). Up to multiple files from different systems.

4

Schema Inference Agent

Claude reads each CSV, identifies which columns are case_id, activity, timestamp, actor, system references. Returns confidence-scored schema inference.

5

Case Correlation Engine

Links events across systems using: explicit key matching → customer spine + temporal proximity → cross-reference field detection. Builds unified EventLog in DB.

6

Process Mining + APA Opportunities

Claude analyses the EventLog for bottlenecks, rework loops, cycle time variance, dark process zones, process variants. Outputs APA automation recommendations per step.
